VS大量报错

发表于 2024-05-10 更新于 2024-09-18 分类于 后端

兄弟们，我简直无语了。最近一觉醒来，发现我的 visual stdio2022 出现大量报错，原本写好的程序全部报错，打不开各个头文件，这个 bug 简直要了我的老命。去网上搜了搜教程各种五花八门的操作都有，什么包含头文件之类的，教你用重定向设置版本的。我都试过了没用，不过到重定向设置版本再去运行的时候，有了一个报错，这是一个好消息，至少终于知道问题在哪里了，之后在我的一番折腾，把 vs 删了又卸载了 20 多次终于弄好了。
接下来我给大家说一下问题吧，问题出在你最近格式化磁盘，还有清理电脑垃圾的时候清理了依赖项包。突然报错的原因是因为系统本身的内置缓存区会使用你在没删除之前的配置，当年删除后，顶多运行几个程序就出现大量报错。而且你再无论怎么安装卸载都不管用，你登陆了 VS 账号，他会保存你所有设置，包括你删除修改的设置，即使你使用安装里面的修复也没用，当你安装了重定向的版本 sdk 再去属性里配置也没用，他还是会告诉你找不到。废话不多说，直接上解决办法。
解决 1：找到如下图片的位置，一般默认在你安装的公布各项插件的磁盘里，把配置改成我的这个配置

然后保存重启 vs.

2：彻底卸载 vs 然后开发工具选所有和 c 有关的，如果你不是就选择和其他语言有关的，因为我是 c。（多说一句，当你的文件配置误删的话，就做好之前项目没有的准备，一切都要从 0 开始），然后选择你电脑 wind 的版本对应的 sdk，例如 win11 就下载 win11sdk。然后在项目属性里重新定向版本，如果还是找不到，就用上面的办法

3：彻底没办法怎么弄都弄不好，没有关系，重新去注册一个 VS 账号，把 VS 清理残余彻底一点，拿来直接就好。

4：要是你不想注册也不想删除，电脑空间也不是很大，也没太多内容，直接去备份重装系统就好（这是最最最坏的办法），不到万不得已不要用这个方法，万一没好，得不偿失。

笑傲博客风云

发表于 2024-05-07

误入 “博” 途，一切从零开始

话说那天，阳光明媚，我误入江湖，踏入这片神秘的土地 ——“博客”。是的，你没听错，我就是一个博客新手，刚刚踏上这片充满奇遇和挑战的江湖。在此之前，我对博客的了解几乎为零。但我相信，无知者无畏，我定能在这片土地上闯出一片天地！

博客新手，崭露锋芒！！

身为新手，当然要了解一下博客的世界。博客，又称网络日志，是人们分享心得、交流思想的平台。在这里，我可以畅所欲言，讲述生活中的点点滴滴，也可以结识志同道合的朋友，共同探讨人生百态。那么，作为一个新手，我要如何在博客这个世界里立足呢？答案就是：笑对人生百态，用心去感悟这个世界！

探秘博客世界，笑对人生百态

初入博客，我被这里的繁华所吸引。各式各样的博客，犹如江湖中的武林高手，各具特色。有的幽默风趣，让人忍俊不禁；有的深入浅出，让人受益匪浅；有的激情四溢，让人热血沸腾。而我，也将成为这个江湖中的一员，带着我的笑容和故事，笑对人生百态。

为何选择它，听听我的故事

它为何如此魅力无穷？为何我能毅然决然地投身于这个江湖？这一切，都要从我的一个梦想说起。我梦想着，有一天，我的文字能够温暖他人的心灵；我梦想着，有一天，我的故事能够激励更多的人。而博客，正是实现这个梦想的最佳舞台。在这里，我可以畅所欲言，让我的文字和故事传遍四海。

博客江湖，我来也！

俗语说：“初生牛犊不怕虎。” 作为一个博客新手，我无畏无惧，勇往直前。我将带着我的笑容和故事，闯荡这个博客江湖，与各位高手过招，共同成长。让我们一起，笑傲博客风云！

最后，我要感谢每一位阅读这篇文章的朋友。你们的关注和支持，是我前行的动力。在未来的日子里，让我们一起，笑对人生百态，共同书写博客江湖的精彩篇章！
